## Project

**AssistFlow Landing Page**

A high-converting landing page hero section copy and structure for AssistFlow, an autonomous AI customer support desk SaaS tailored for startups. The hero section includes outcome-focused headlines, direct subheadlines, statistical proof points, and clear call-to-actions (CTAs).

**Core Value:** Convert visiting startup founders by communicating the value of autonomous, hands-off customer support with zero corporate speak.

### Constraints

- **Headline Length**: Max 8 words — to keep it readable and punchy.
- **Subheadline Length**: Max 20 words — to ensure conciseness.
- **Tone**: Direct, startup-founder-friendly, zero corporate speak — to build trust with the target audience.
- **Structure**: Each variant must contain 1 headline, 1 subheadline, 3 statistical proof points, and 2 CTA labels.

## Technology Stack

## Recommended Stack

### Core Technologies

| Technology | Version | Purpose | Why Recommended |
|------------|---------|---------|-----------------|
| HTML5 | Living Standard | Document Structure | Native, fast semantic layout for landing pages |
| Vanilla CSS | CSS3 | Styling & Layout | Maximum control, flexibility, performance, and custom design aesthetics |
| Google Fonts | API v2 | Typography | Load premium fonts like Outfit or Inter to make the page feel modern and polished |
| Markdown | GFM | Copy Drafting | Standard format for writing, reviewing, and iterating on landing page copy |

### Supporting Libraries

| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| Lucide Icons | Latest | Visual UI Cues | Adding clean, scalable SVG icons to CTA buttons and proof points |
| Hemingway App | web | Copy Readability | Verifying the copy tone is direct and free of passive/complex sentences |

### Development Tools

| Tool | Purpose | Notes |
|------|---------|-------|
| Live Server | Quick iteration feedback | Instantly see styling and copy changes in the browser |
| WebP / SVG | Visual assets format | High-quality, fast-loading images and icons for the demo section |

## Alternatives Considered

| Recommended | Alternative | When to Use Alternative |
|-------------|-------------|-------------------------|
| Vanilla CSS | Tailwind CSS | If rapid UI prototyping is preferred and the user is comfortable with utility classes |
| Google Fonts | System Fonts | If absolute zero third-party network requests are required |

## What NOT to Use

| Avoid | Why | Use Instead |
|-------|-----|-------------|
| Heavy Frameworks (Next.js/React) | Overkill for a static marketing landing page copy and mockup | Single index.html + index.css |
| Corporate Buzzwords | Triggers developer skepticism and reduces conversion | Clear, metric-based proof points |
